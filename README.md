## Project constraints and methodology

### Imposed constraints

Goal of this exercise was to make us assimilate all core concepts of HTML and CSS as such we had to respect these.

- No third-party framework
- No backend
- No IA generation

### Project methodology
To ensure we shared the vision and limit conflicts when pushing code we enforced a few simple rules throughout the duration.

1. We worked together on the "page template" (sections structure + applied CSS) then each took care of "filling in" different pages.
2. Whenever we modified "common code" we did not push directly on dev but made a Pull Request from "personal branch", which had to be checked and approved by peer: this allowed fresh eyes to view code and detect potential flaws while also making reviewer understand and "learn" about peer's code naturally.
3. We checked our HTML structure against basic "HTML standard code validator" and content against grammar/orthograph checkers.
4. Only once finalized and validated were pages pushed to DEV.

We also used Github's tickets and Wiki scarcely, as we realized a few days in we didn't need it as our communication and collaboration workflow was working fine without them.
